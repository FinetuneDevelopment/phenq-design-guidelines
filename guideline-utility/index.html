Border radius

Sizes

Border radius has three different sizes:

radius-big

Generally used on call-to-action panels which span the width of the page, such as bleed bisect.

radius-medium

Used on call-to-action panels within a grid layout, such as this example here.

radius-small

Vertical spacing

Vertical spacing is achieved using four different classes:

gutter-large

This adds a large space under elements. It is mostly used on the home page.

gutter-medium

This can be added to each discrete block of content which takes up the full width of the page.

gutter-small

This is sometimes useful within blocks, for adding a small amount of whitespace between blocks of similar content, such as different rows within the same section.

flush

This removes any pre-existing margin bottom from an element.

  • text-uppercase converts text to uppercase
  • text-breath adds a little bit of letter-spacing than usual to the text. This is primarily used for certain headings or legal text which is rarely used across the site.

Quick layouts

Space between

Using the space-between class, a quick navbar-like layout can be achieved. It takes a number of elements and spaces them out within their container, wrapping onto a second line, if required.

Button toolbars

By default, buttons have margins which push against each other. Using a toolbar class allows buttons to sit closer together than they usually would.

Default layout

Toolbar layout

Flex-vert

Adding a flex-vert class onto an element inside a Bootstrap column (or added directly to the column itself) gives it a vertical flexbox layout. This means that the last child of a flex-vert class can have a mt-auto class applied to it, which will align those elements to the bottom of the column.

If a cta-box class is used inside the Bootstrap column then the flex-vert class should be applied to that element and not the parent column.

Accessibility

Hiding text for screen readers

Elements can be hidden for all browsers other than speech readers using a sr-only class. For example, in the below call-to-action, the phrase Find out more is not accessible as it does not uniquely identify the target of the link:

Find out more

A more accessible version of this link might look like this:

Find out more about the twelve eating habits for weight loss

This extra information can be conveyed only to users of screen readers by wrapping it in a sr-only class:

Find out more about the twelve eating habits for weight loss

It is appropriate to hide some elements from all users. For example, the arrows which control scrollable elements only appear when there is enough space for the elements to scroll:

When there is nothing for these controls to do, a class of hidden is added to them with JavaScript, which hides them from all users.

Showing elements on focus

Sometimes, it may be required to hide an element until it falls into focus. For example, the currency selector has a submit button which is hidden for users who have enabled JavaScript, but which still traps focus. For this reason, the button appears for the benefit of users who are navigating using the keyboard.

Adding a class of kb-focus in conjunction with sr-only allows elements to be hidden until they fall into focus.
